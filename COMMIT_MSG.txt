feat: migrate media storage to Cloudinary

This commit completes the migration from local file storage to Cloudinary for all media assets.

Key changes:
- Added Cloudinary SDK and configuration
- Created CloudinaryService for centralized operations with robust error handling
- Added client-side utilities for Cloudinary URL transformations
- Enhanced MediaUploader with progress tracking and optimized previews
- Implemented responsive images and video thumbnails
- Removed special CORS handling for media files
- Added verification and migration scripts
- Created comprehensive documentation

This change resolves the CORS issues with video playback and provides better
scalability for media management through Cloudinary's CDN and transformation capabilities.
The implementation follows best practices for media handling in web applications.
